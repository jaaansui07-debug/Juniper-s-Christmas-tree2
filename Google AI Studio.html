“项目根目录”（Project Root Directory）就是你存放这个项目所有代码的最外层文件夹。

简单来说，就是你打开电脑，新建的那个文件夹（例如你可能把它命名为 christmas-tree），在这个文件夹里，你可以看到 package.json、index.html、vite.config.ts 这些核心文件。

如何创建 public 文件夹并放入文件

请按照以下步骤操作，非常简单：

1. 找到位置

打开你的项目文件夹（根目录）。你应该能看到 src 文件夹和 index.html 文件。

2. 新建文件夹

在根目录下的空白处右键 -> 新建文件夹 -> 命名为 public（注意全是小写）。

3. 放入文件

将你下载好的 hand_landmarker.task 文件直接拖进去。

最终的文件结构图

弄好之后，你的文件夹结构应该长这个样子（注意看 public 文件夹的位置）：

code Text
    christmas-tree/          <-- 这就是【根目录】
├── node_modules/        (如果你运行了 npm install 才会生成)
├── public/              <-- 【你刚刚新建的文件夹】
│   └── hand_landmarker.task  <-- 【把你下载的文件放这里】
├── src/                 (存放 App.tsx, main.tsx 的地方)
│   ├── components/
│   ├── types.ts
│   └── ...
├── index.html           (网页入口文件)
├── package.json         (项目配置文件)
├── tsconfig.json
└── vite.config.ts

为什么这么做？

在 Vite（这个项目使用的构建工具）中，public 文件夹是一个特殊的地方。

  • 放在 src 里的文件会被打包、压缩、混淆。

  • 放在 public 里的文件不会被修改，构建时会自动复制到根路径下。

这就是为什么我们在代码里写 modelAssetPath: '/hand_landmarker.task'（斜杠开头），程序就能直接在这个特殊的文件夹里找到它，从而避免去连接被屏蔽的 Google 服务器。
